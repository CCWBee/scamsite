## Mobile-First Approach

### What is Mobile-First?

Mobile-first means writing base styles for mobile devices, then adding complexity for larger screens. This approach:

- Ensures mobile users get a fast, optimized experience
- Keeps CSS simpler and more maintainable
- Forces prioritization of essential content
- Aligns with Google's mobile-first indexing

### How It Works

Base styles (no prefix) apply to **all screen sizes**. Prefixed styles apply at that breakpoint **and above**.

```jsx
// Mobile-first example
<div className="
  text-sm        /* Base: applies to all sizes (mobile) */
  md:text-base   /* Medium screens (768px+): larger text */
  lg:text-lg     /* Large screens (1024px+): even larger */
">
  Responsive text
</div>
```

```jsx
    - md (768px+): max-width 768px with 32px padding
    - lg (1024px+): max-width 1024px with 32px padding
    - xl (1280px+): max-width 1200px (our custom max) with 32px padding
  */}
  Content constrained to readable width
</div>
```

## Using Breakpoints in JavaScript

For conditional logic in React components:

```typescript
import { breakpoints } from '@/lib/design-tokens';

// Check screen width
const isTablet = window.innerWidth >= parseInt(breakpoints.md);
const isDesktop = window.innerWidth >= parseInt(breakpoints.lg);

// With React hook
import { useState, useEffect } from 'react';

function useBreakpoint(breakpoint: keyof typeof breakpoints) {
  const [matches, setMatches] = useState(false);

  useEffect(() => {
    const query = window.matchMedia(
      `(min-width: ${breakpoints[breakpoint]})`
    );
    setMatches(query.matches);

    const listener = (e: MediaQueryListEvent) => setMatches(e.matches);
    query.addEventListener('change', listener);
    return () => query.removeEventListener('change', listener);
  }, [breakpoint]);

  return matches;
}

// Usage
function MyComponent() {
  const isDesktop = useBreakpoint('lg');

  return isDesktop ? <DesktopView /> : <MobileView />;
}
```

## Best Practices

### Do

- Start with mobile styles, add complexity for larger screens
- Test designs at each breakpoint
- Use semantic breakpoint names (`md`, `lg`) not pixel values
- Consider touch targets on mobile (minimum 44x44px)
- Ensure text remains readable at all sizes

### Don't

- Write desktop styles first and "shrink" for mobile
- Use too many breakpoint overrides (keep it simple)
- Forget to test on real devices, not just browser resize
- Assume everyone has a large screen
- Ignore intermediate sizes between breakpoints

### Testing Checklist

- [ ] **320px** - Smallest supported mobile width
- [ ] **375px** - iPhone SE, common mobile size
- [ ] **414px** - iPhone Plus/Max sizes
- [ ] **768px** - iPad portrait, tablet breakpoint
- [ ] **1024px** - iPad landscape, small desktop
- [ ] **1280px** - Common laptop resolution
- [ ] **1440px** - Large desktop monitors
- [ ] **1920px** - Full HD monitors

## Key Breakpoints for ScamAware Jersey

Based on the project requirements:

| Breakpoint | Width | Primary Use |
|------------|-------|-------------|
| Mobile (default) | 320px minimum | Single column, stacked layout |
| Tablet (`md`) | 768px | Two-column layouts, expanded nav |
| Desktop (`lg`) | 1024px | Full navigation, sidebar options |
| Large (`xl`) | 1440px | Optimized wide layouts |

These align with the design specification's mobile-first approach with breakpoints at 320px, 768px, 1024px, and 1440px.
