# Spacing

The ScamAware Jersey spacing system is based on a 4px (0.25rem) base unit, ensuring consistent visual rhythm throughout the application. This scale provides predictable, harmonious spacing for margins, padding, and gaps.

---

## Usage Guidelines

### When to Use Each Spacing Value

| Scale | Pixels | Use Case |
|-------|--------|----------|
| `0.5-1` | 2-4px | Icon gaps, tight inline elements |
| `2` | 8px | Button padding (small), tight gaps |
| `3-4` | 12-16px | Default padding for small components, input fields |
| `5-6` | 20-24px | Card padding, comfortable component spacing |
| `8` | 32px | Section padding (mobile), card gaps |
| `10-12` | 40-48px | Section margins, vertical rhythm |
| `16-20` | 64-80px | Page section spacing |
| `24` | 96px | Hero section padding, major page breaks |

## Responsive Spacing

Spacing should adapt to screen size. Use responsive modifiers for optimal layouts.

### Mobile-First Approach

```jsx
// Padding increases at larger screens
<section className="p-4 md:p-6 lg:p-8">
  Content with responsive padding
</section>

// Gap increases at larger screens
<div className="flex flex-col gap-4 md:gap-6 lg:gap-8">
  <Card />
  <Card />
  <Card />
</div>

// Margin increases at larger screens
<div className="mt-8 md:mt-12 lg:mt-16">
  Section with responsive margin
</div>
```

### Responsive Spacing Patterns

| Element | Mobile | Tablet (md) | Desktop (lg) |
|---------|--------|-------------|--------------|
| Page padding | `p-4` (16px) | `p-6` (24px) | `p-8` (32px) |
| Section margin | `my-8` (32px) | `my-12` (48px) | `my-16` (64px) |
| Card gap | `gap-4` (16px) | `gap-6` (24px) | `gap-8` (32px) |
| Container padding | `px-4` (16px) | `px-6` (24px) | `px-8` (32px) |

## Special Spacing Tokens

### Component-Specific Values

```typescript
// Design token values for specific components
spacing: {
  'header': '4.5rem',  // 72px - Header/navigation height
  'footer': '15rem',   // 240px - Footer height
  '18': '4.5rem',      // 72px - Page section padding
  '22': '5.5rem',      // 88px - Extra large section padding
}
```

### Usage Example

```jsx
// Header-aware content positioning
<main className="pt-header">
  Main content with header offset
</main>

// Section with custom spacing
<section className="py-18 lg:py-22">
  Large section content
</section>
```

## Container Configuration

The container has built-in responsive padding:

```typescript
container: {
  center: true,
  padding: {
    DEFAULT: '1rem',   // 16px on mobile
    sm: '1.5rem',      // 24px on small screens
    md: '2rem',        // 32px on medium and up
    lg: '2rem',
    xl: '2rem',
  },
  maxWidth: '1200px', // Primary content width
}
```

```jsx
// Centered container with responsive padding
<div className="container mx-auto">
  Content automatically gets responsive padding
</div>

// Manual container with max-width
<div className="max-w-content mx-auto px-4 md:px-6 lg:px-8">
  Content with max-width-content (1200px)
</div>
```

## Best Practices

### Do

- Use consistent spacing values from the scale
- Increase spacing for larger, more important sections
- Apply responsive spacing for different screen sizes
- Use `gap` for flex/grid layouts instead of margins

### Don't

- Mix arbitrary pixel values with the spacing scale
- Use tight spacing for touch targets (minimum 44px)
- Forget to adjust spacing for mobile devices
- Overuse large spacing values in compact areas
